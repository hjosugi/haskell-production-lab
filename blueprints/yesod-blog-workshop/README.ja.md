# Yesodブログワークショップ設計図

この設計図は、GREEスタイルの内部チュートリアルのアイデアを模倣しています：テンプレートから始めて、フォームを追加し、その後Persistent、Aeson、認証を追加します。

## 推奨ステップ

1. シングルファイルのYesodアプリ
2. ブログルートとHamletテンプレート
3. フォーム送信
4. Persistentモデル
5. Aesonを使ったJSONエンドポイント
6. 認証のスケルトン
7. デプロイメントチェックリスト

以下のコードは、通常のCabalビルドから意図的に分離しています。なぜなら、Yesodプロジェクトのスキャフォールディングは通常、自身のパッケージと設定を持つためです。
