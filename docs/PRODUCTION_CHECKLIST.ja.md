# 本番チェックリスト

## ビルドとリリース

- [ ] `cabal build all` が成功する
- [ ] `cabal test all` が成功する
- [ ] Haddock がビルドされる
- [ ] Docker イメージがビルドされる
- [ ] プルリクエスト時に CI が実行される
- [ ] リリースアーティファクトが再現可能である

## 実行時

- [ ] `/health` がサービス名とバージョンを返す
- [ ] `/metrics` がスクレイプ可能である
- [ ] ログにタイムスタンプ、レベル、相関IDが含まれる
- [ ] グレースフルシャットダウンが実装されている
- [ ] ワーカーのリトライ回数に制限がある
- [ ] キューにデッドレター処理がある

## データ

- [ ] データベースのマイグレーションがバージョン管理されている
- [ ] バックアップがテストされている
- [ ] 個人情報（PII）フィールドが分類されている
- [ ] 金銭移動には冪等性キーが必要
- [ ] イベントログのスキーマは追記専用である

## セキュリティ

- [ ] 書き込みエンドポイントに認証・認可が追加されている
- [ ] リクエストサイズの制限が追加されている
- [ ] 入力検証が構造化されたエラーを返す
- [ ] シークレットがコミットされていない
- [ ] 依存関係の監査がCIの一部として行われている

## 可観測性

- [ ] エンドポイントとステータスごとのメトリクス
- [ ] 構造化されたJSONログ
- [ ] サービス間のトレース
- [ ] エラー率、レイテンシ、キュー深さのアラートルール
